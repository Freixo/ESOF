Aplicações Nativas

  • Desenvolvidas para plataformas específicas
  • Distribuídas através de lojas virtuais
    • Play Store, Windows Store, App Store
  • Instaladas diretamente no diapositivo do utilizador

Aplicações HTML5

  • Correm em qualquer plataforma
    • Browser moderno - implemente os standarts mais recentes
  • Existem na web e são acedidas através do browser do diapositivo móvel do utilizador


“Write once, run anywhere”

Aplicações Híbridas

  • Desenvolvidas sobre uma base de código nativo
  • Utilizam o motor do browser do OS para correr HTML e processar JavaScript localmente, que constitui a aplicação

Tendências de mercado

  • Os utilizadores preferem aplicações nativas
    • Melhor performance, usabilidade e segurança
  • Os developers/publishers preferem aplicações HTML5
    • Adaptação a qualquer OS, lucros totais para os criadores e só necessita de uma aprovação

Aplicações HTML5 vs Nativas

O que varia!? Uma aplicação nativa, como é instalada directamente no dispositivo sem ligação à internet. As aplicações nativas tornam-se mais caras do que as HTML5, uma vez que é necessário desenvolver várias para cada plataforma, em vez de apenas uma, que funciona em todas as plataformas.

Experiêcia do utilizador e desempenho

Apps nativas: Mais rápidas e interactivas

HTML5 Web Apps: Dependem da ligação a Internet

Aplicações html5 não tem tanta integração com o OS

Monetização

As aplicações nativas são fáceis de rentabilizar devido as Lojas de Aplicações, por outro lado as Web Apps tem dificuldade de descoberta, falta de modelos de anúncio,...

Dificuldade de descoberta Limitações de suporte Falta de modelos de anúncios Hesitação pelos utilizadores Inexistência de modelos de pagamento

Custo e facilidade de desenvolvimento

Aplicações nativas: maior investimento e tempo de desenvolvimento

Linguagens diferentes para cada OS (Java, Objective C, .NET)

Aplicações HTML5: menor investimento e mais fácil

HTML5, CSS e JavaScript

Fragmentação

Apps Nativas: necessidade de suportar multiplas plataformas

Web Apps: necessitam de suportar diferentes browsers, com diferente suporte as versões do HTML.

Updates e controlo de distribuição

As Apps HTML5 como não são instaladas são actualizadas no momento do update.

As rapidez de updates das apps nativas dependem da loja de distribuição

Dependência de funcionalidades do SO/Hardware

Apps Nativas

  • Podem utilizar funcionalidades de hardware
    • Acelarómetro, GPS, Câmara, Giroscópio, ...
  • Podem utilizar funcionalidades do SO
    • Touch events, integração com o ambiente do SO

Apps HTML5

  • Dependem da interface do browser
  • Muitas limitações na utilização de hardware

Segurança

Web Apps

  • Facilidade de ataque
    • cross-site scripting
    • cookie manipulation
    • sql injection
  • Dados em cache não encriptados


Apps Nativas

  • Mais difíceis de atacar
  • Mais fáceis de tornar seguras
